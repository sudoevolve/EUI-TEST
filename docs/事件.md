# 事件系统

事件系统位于 `core/event.h`。当前是轻量 pointer 事件和交互状态封装，真正的事件更新由 `core::dsl::Runtime` 统一执行。

## PointerEvent

```cpp
struct PointerEvent {
    double x;
    double y;
    double deltaX;
    double deltaY;
    bool down;
};
```

含义：

- `x / y`：当前鼠标坐标。
- `deltaX / deltaY`：与上次读取的差值。
- `down`：鼠标左键是否按下。

读取方式：

```cpp
core::PointerEvent event = core::readPointerEvent(window, pointerScale);
```

`pointerScale` 用来把 GLFW window 坐标转换成 framebuffer 坐标。

## pointerScale 与 dpiScale

`pointerScale`：

- 来源：`framebufferSize / windowSize`
- 用途：鼠标坐标转换和 hit-test

`dpiScale`：

- 来源：`glfwGetWindowContentScale`
- 用途：UI 尺寸、字号、圆角、阴影等逻辑值转像素

两者不一定相同。

## InteractionState

```cpp
struct InteractionState {
    bool hover;
    bool pressed;
    bool clicked;
    bool drag;
    bool changed;
    double dragStartX;
    double dragStartY;
    double dragDeltaX;
    double dragDeltaY;

    void update(const Rect& bounds, const PointerEvent& event);
};
```

状态含义：

- `hover`：指针是否在 bounds 内。
- `pressed`：是否在元素上按下。
- `clicked`：本帧是否完成点击。
- `drag`：是否正在拖动。
- `changed`：交互状态是否变化。

## 点击判定

当前 click 判定：

```cpp
clicked = hover && pressed && !event.down;
```

即：

1. 鼠标在元素上。
2. 前一状态是 pressed。
3. 当前鼠标松开。

## Hover / Pressed

```cpp
hover = bounds.contains(event.x, event.y);
pressed = hover && event.down;
```

当前 pressed 只在 hover 内按住时为 true。按下后拖出元素会变 false。

## Drag

当前 drag 判定：

```cpp
drag = pressed && (abs(dragDeltaX) > 2.0 || abs(dragDeltaY) > 2.0);
```

阈值是 2 像素，避免轻微抖动被判定为 drag。

## DSL 中的事件写法

交互目前挂在 `Rect` 上：

```cpp
ui.rect("hit")
    .size(240.0f, 70.0f)
    .states(normal, hover, pressed)
    .onClick([] {
        // update app state
    })
    .build();
```

`states(...)` 会设置：

- normal color
- hover color
- pressed color
- `interactive = true`

`onClick(...)` 也会设置 `interactive = true`。

Runtime 每帧会：

1. 读取 pointer event。
2. 遍历所有 `Rect`。
3. 对 interactive rect 做 hit-test。
4. 更新 `InteractionState`。
5. 推进 hover / press 缓动。
6. 如果 clicked，执行 `onClick`。
7. 标记 dirty rect 或请求重新 compose。

## 组件中的事件

组件不要自己持有 `InteractionState`，也不要自己调用 `readPointerEvent`。

正确做法是组件内部声明一个可交互 `Rect`：

```cpp
ui.rect(id + ".bg")
    .states(normal, hover, pressed)
    .onClick(callback)
    .build();
```

例如 `components::button(ui, id)` 内部就是这样做的。按钮组件只组合 DSL，交互由 Runtime 统一处理。

## 事件与动画

Runtime 对 hover / press 使用 `SmoothedValue<float>` 缓动：

- hover 目标值：`1.0` 或 `0.0`
- press 目标值：`1.0` 或 `0.0`

颜色会按 hover/press blend 混合：

```text
normal -> hover -> pressed
```

`visualStateFrom(id + ".bg")` 可以让父容器和子内容共享背景 rect 的 press 状态，用于按钮按压缩放。

示例：

```cpp
ui.stack(id)
    .size(w, h)
    .visualStateFrom(id + ".bg", 0.965f)
    .content([&] {
        ui.rect(id + ".bg")
            .size(w, h)
            .states(normal, hover, pressed)
            .onClick(callback)
            .build();

        ui.text(id + ".text")
            .size(w, h)
            .text("Button")
            .build();
    });
```

## 事件与脏区渲染

交互状态变化不会让整页默认全量重绘。

- hover / press 变化：标记对应 rect 的 dirty rect。
- transition 动画：标记旧视觉区域和新视觉区域的 union。
- clicked 且回调修改状态：`Runtime::needsCompose()` 为 true，app 重新 compose，并保守 full redraw。

## 当前限制

- 只有鼠标左键。
- 没有键盘事件。
- 没有 focus manager。
- 没有 pointer capture。
- 没有事件捕获 / 冒泡。
- 没有命中测试结果链。
- transform 后 hit-test 仍按未变换布局矩形计算。
- 当前只有 `Rect` 可交互，`Text` 不直接接收事件。
