# 组件

组件层位于 `components/`。它不是另一套 UI 系统，也不直接操作 OpenGL primitive。组件的职责是把常用 DSL 结构包装成更短的调用方式。

页面仍然由 `ui.row`、`ui.column`、`ui.stack`、`ui.rect`、`ui.text`、`ui.image` 组成。组件只是帮你少写重复结构和重复样式。

## 当前组件

### panel

`components::panel(ui, id)` 返回一个已经套用默认 theme token 的 `RectBuilder`。默认使用 dark token；需要跟随应用主题时传入 `ThemeColorTokens`：

```cpp
const auto tokens = components::theme::DarkThemeColors();

components::panel(ui, "card", tokens)
    .size(360.0f, 220.0f)
    .build();
```

它本质仍是 `RectBuilder`，所以可以继续链式覆盖 `.color()`、`.border()`、`.shadow()`、`.states()`、`.onClick()`、`.transition()`、`.blur()` 等 rect 能力。

### text / label

`components::text(ui, id)` 返回带默认文本色的 `TextBuilder`。

`components::label(ui, id)` 返回带默认文本色的 label builder。需要跟随应用主题时传入 token：

```cpp
components::text(ui, "title", tokens)
    .size(420.0f, 48.0f)
    .text("EUI Gallery")
    .fontSize(38.0f)
    .color({0.94f, 0.97f, 1.0f, 1.0f})
    .horizontalAlign(core::HorizontalAlign::Center)
    .verticalAlign(core::VerticalAlign::Center)
    .build();
```

还提供 `titleTextStyle(tokens)`、`subtitleTextStyle(tokens)`、`bodyTextStyle(tokens)`，用于快速生成符合 theme token 的 `core::TextStyle`。Text 现在也能使用通用 `.onClick(...)`，但没有内置 hover / pressed 颜色状态。

### image

`components::image(ui, id)` 返回套用默认图片 token 的 `ImageBuilder`。默认使用 dark token；需要跟随应用主题时传入 token：

```cpp
components::image(ui, "logo", tokens)
    .size(120.0f, 120.0f)
    .source("assets/icon.png")
    .cover()
    .build();
```

Image token 目前设置 tint、radius、opacity。Image 支持本地图片、网络图片、SVG 和 Bing daily。它也能通过通用 `.onClick(...)` 接收点击；需要 hover/pressed 视觉反馈时建议外层叠 rect。

### theme tokens

`components/theme.h` 移植了参考项目 `ThemeTokens.h` 中可直接复用的一套配色和视觉 token，不依赖参考项目里的 Renderer 或 UIPrimitive。

```cpp
#include "components/theme.h"

const auto colors = components::theme::DarkThemeColors();
const auto page = components::theme::pageVisuals(colors);

ui.rect("card")
    .size(360.0f, 220.0f)
    .color(page.cardColor)
    .border(1.0f, colors.border)
    .build();
```

当前包含：

- `ThemeColorTokens`：background、primary、surface、surfaceHover、surfaceActive、text、border、dark。
- `PageVisualTokens`：页面标题、正文、卡片、间距、字号和 section 尺寸 token。
- `UIFieldVisualTokens`：输入类控件圆角、内边距、焦点线、弹层阴影 token。
- `LightThemeColors()` / `DarkThemeColors()`：参考项目同名入口。
- `resolveFieldFill(...)` / `ResolveFieldFill(...)`：按 hover / active 混合字段背景色。
- `border(...)`、`popupShadow(...)`：常用样式辅助函数。

### button

`components::button(ui, id)` 是一个薄 DSL builder。内部组合：

- `ui.stack(id)`：按钮根节点。
- `ui.rect(id + ".bg")`：背景、hover、pressed、点击区域。
- `ui.row(id + ".content")`：图标和文字布局。
- `ui.text(id + ".icon")`：可选图标。
- `ui.text(id + ".text")`：文字。

常用写法：

```cpp
components::button(ui, "save")
    .size(180.0f, 54.0f)
    .theme(tokens)
    .icon(0xF0C7)
    .text("Save")
    .transition(0.18f)
    .onClick([] {
        // update app state
    })
    .build();
```

按钮默认 `ButtonStyle` 来自 theme token，也可以用 `.theme(tokens)`、`.primaryTheme(tokens)`、`.secondaryTheme(tokens)` 切换。hover / pressed 视觉变化由内部背景 rect 的 `.states(normal, hover, pressed)` 完成。点击由 Runtime 的统一事件系统处理。

## 组件写法原则

- 简单组件返回 core builder，并先套用 theme token 默认样式，例如 `panel`、`text`、`image`。
- 复杂组件提供一个薄 builder，内部只声明 DSL 树。
- 组件 id 必须稳定，内部子节点使用 `id + ".name"`。
- 交互使用 DSL 的 `.interactive()`、`.onClick()`、`.states()`。
- 需要按压缩放时，用外层 `.visualStateFrom(interactiveRectId, pressedScale)`。
- 不要在组件里 new / initialize / render / destroy primitive。
- 不要在组件里自己读鼠标事件。
- 不要在组件里维护 hover / pressed 状态。
- 不要绕过 `core::dsl::Runtime`。

推荐结构：

```cpp
class CardButtonBuilder {
public:
    CardButtonBuilder(core::dsl::Ui& ui, std::string id)
        : ui_(ui), id_(std::move(id)) {}

    CardButtonBuilder& text(const std::string& value) {
        text_ = value;
        return *this;
    }

    CardButtonBuilder& onClick(std::function<void()> callback) {
        onClick_ = std::move(callback);
        return *this;
    }

    void build() {
        ui_.stack(id_)
            .size(260.0f, 72.0f)
            .visualStateFrom(id_ + ".hit", 0.965f)
            .content([&] {
                const core::Color bg = tokens_.surface;
                ui_.rect(id_ + ".hit")
                    .size(260.0f, 72.0f)
                    .states(bg, components::theme::buttonHover(tokens_, bg), components::theme::buttonPressed(tokens_, bg))
                    .radius(14.0f)
                    .border(components::theme::buttonBorder(tokens_, false))
                    .transition(0.16f, core::Ease::OutCubic)
                    .onClick(onClick_)
                    .build();

                ui_.text(id_ + ".label")
                    .size(260.0f, 72.0f)
                    .text(text_)
                    .color(tokens_.text)
                    .horizontalAlign(core::HorizontalAlign::Center)
                    .verticalAlign(core::VerticalAlign::Center)
                    .build();
            })
            .build();
    }

private:
    core::dsl::Ui& ui_;
    std::string id_;
    std::string text_ = "Button";
    std::function<void()> onClick_;
    components::theme::ThemeColorTokens tokens_ = components::theme::DarkThemeColors();
};
```

重点是：组件没有自己的渲染对象，也没有自己的事件循环。组件只是把 DSL 固定成更好用的调用形式。

## 脏区渲染关系

组件天然走 DSL Runtime，所以正常组件都会进入当前脏区渲染流程。

Runtime 会处理：

- hover / pressed 状态变化。
- transition 动画推进。
- rect frame / color / opacity / radius / border / shadow / blur / transform。
- text frame / color / opacity。
- image frame / tint / opacity / radius / transform。
- framebuffer cache + scissor 局部重绘。

脏区能力不是组件自己实现的。组件只要稳定声明 DSL 树，Runtime 才能根据 id 和状态缓存做动画、命中测试和 dirty rect。

## 当前限制

- 还没有 clip / scroll。
- 还没有 z-index，绘制和命中测试顺序都跟声明顺序相关，后声明的 interactive 元素会成为更上层命中对象。
- transform 后的 hit-test 仍按布局矩形计算。
- 组件没有生命周期钩子；状态应放在页面或业务层，再通过 compose 写回 DSL。
