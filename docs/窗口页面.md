# 窗口与页面

当前项目的窗口、页面和 DSL App 已经分层：

- `main.cpp`：创建 GLFW 窗口、OpenGL 上下文、主循环、DPI 和事件等待。
- `app/app.h`：定义应用入口函数。
- `app/dsl_app.h`：把 DSL Runtime 接到应用生命周期。
- `app/*.cpp`：每个 cpp 是一个独立 demo / 页面程序。
- `CMakeLists.txt`：为每个 `app/*.cpp` 生成一个可执行程序。

页面开发通常只需要写 `app/*.cpp`，不需要直接改窗口循环。

## App 入口

使用 DSL 的 app 文件需要包含：

```cpp
#include "app/dsl_app.h"
```

然后实现两个函数：

```cpp
namespace app {

const DslAppConfig& dslAppConfig();
void compose(core::dsl::Ui& ui, const core::dsl::Screen& screen);

} // namespace app
```

最小示例：

```cpp
#include "app/dsl_app.h"

namespace app {

const DslAppConfig& dslAppConfig() {
    static const DslAppConfig config = {
        "My DSL App",
        "my_app",
        {0.07f, 0.08f, 0.10f, 1.0f},
        1280,
        800
    };
    return config;
}

void compose(core::dsl::Ui& ui, const core::dsl::Screen& screen) {
    ui.stack("root")
        .size(screen.width, screen.height)
        .align(core::Align::CENTER, core::Align::CENTER)
        .content([&] {
            ui.rect("card")
                .size(360.0f, 220.0f)
                .color({0.10f, 0.12f, 0.16f, 1.0f})
                .radius(18.0f)
                .build();

            ui.text("title")
                .size(360.0f, 48.0f)
                .text("Hello EUI")
                .fontSize(32.0f)
                .color({0.94f, 0.97f, 1.0f, 1.0f})
                .horizontalAlign(core::HorizontalAlign::Center)
                .verticalAlign(core::VerticalAlign::Center)
                .build();
        })
        .build();
}

} // namespace app
```

## DslAppConfig

`DslAppConfig` 当前字段：

```cpp
struct DslAppConfig {
    const char* title = "DSL App";
    const char* pageId = "app";
    core::Color clearColor = {0.16f, 0.18f, 0.20f, 1.0f};
    int windowWidth = 800;
    int windowHeight = 600;
};
```

- `title`：窗口标题。
- `pageId`：Runtime compose 页面时使用的页面 id，保持稳定。
- `clearColor`：窗口清屏色，也是 Runtime 离屏缓存重建时使用的背景色。
- `windowWidth` / `windowHeight`：初始窗口大小。

`pageId` 不要每帧变化。它用于 Runtime 区分页面缓存，频繁变化会导致更多重建和重绘。

## compose

`compose(core::dsl::Ui& ui, const core::dsl::Screen& screen)` 是页面声明入口。

`screen.width` 和 `screen.height` 是逻辑尺寸，已经由 `app/dsl_app.h` 根据 DPI scale 换算过。页面布局应当使用这个逻辑尺寸，不要直接取 framebuffer 像素尺寸。

推荐根节点写法：

```cpp
void compose(core::dsl::Ui& ui, const core::dsl::Screen& screen) {
    ui.row("root")
        .size(screen.width, screen.height)
        .content([&] {
            // sidebar
            // content
        })
        .build();
}
```

页面状态可以放在 `app/*.cpp` 的匿名 namespace 中：

```cpp
namespace {

int selectedPage = 0;
bool optionMotion = true;

} // namespace
```

点击事件修改状态后，`app/dsl_app.h` 会在 Runtime 需要重新 compose 时重新声明页面，并标记一次 full redraw。

## 主循环行为

`main.cpp` 当前负责：

- 初始化 GLFW。
- 创建 OpenGL 3.3 core profile 窗口。
- 加载 glad。
- 调用 `app::initialize(window)`。
- 每帧计算 `deltaSeconds`。
- 读取 framebuffer 尺寸、DPI scale、pointer scale。
- 调用 `app::update(...)`。
- 只有需要渲染时才调用 `app::render(...)` 和 `glfwSwapBuffers(...)`。
- 动画中使用 `glfwPollEvents()`，没有动画时使用 `glfwWaitEvents()`。
- 按 `Esc` 关闭窗口。
- 退出时调用 `app::shutdown()`。

这意味着普通静态页面不会一直满帧刷新。hover、pressed、transition 动画会让 Runtime 请求渲染；动画停止后会回到等待事件。

## dsl_app.h 做了什么

`app/dsl_app.h` 是 DSL App 的适配层，封装了：

- `app::windowTitle()`
- `app::initialWindowWidth()`
- `app::initialWindowHeight()`
- `app::initialize(GLFWwindow*)`
- `app::update(...)`
- `app::isAnimating()`
- `app::render(...)`
- `app::shutdown()`

内部持有一个静态 `core::dsl::Runtime`：

```cpp
inline core::dsl::Runtime& dslRuntime() {
    static core::dsl::Runtime runtime;
    return runtime;
}
```

`update` 中会先 compose 当前页面，再把鼠标、时间、DPI 信息交给 Runtime：

```cpp
detail::dslRuntime().compose(config.pageId, logicalWidth, logicalHeight, ...);
const bool changed = detail::dslRuntime().update(window, deltaSeconds, pointerScale, dpiScale);
```

如果 Runtime 判断需要重新 compose，会再次 compose，并标记 full redraw：

```cpp
if (detail::dslRuntime().needsCompose()) {
    composeFrame();
    detail::dslRuntime().markFullRedraw();
}
```

## Demo 程序

当前 `app/` 下主要有：

- `demo.cpp`：简单 DSL 示例。
- `gallery.cpp`：完整 Gallery，包含 Controls / Text / Animation / Settings / About。

`CMakeLists.txt` 会扫描 `app/*.cpp`：

```cmake
file(GLOB APP_SOURCES CONFIGURE_DEPENDS "${CMAKE_CURRENT_SOURCE_DIR}/app/*.cpp")
foreach(app_source ${APP_SOURCES})
    get_filename_component(app_name "${app_source}" NAME_WE)
    add_executable(${app_name} main.cpp "${app_source}" ${CORE_SOURCES})
    configure_opengl_app(${app_name})
endforeach()
```

所以新增 `app/example.cpp` 后，会生成 `example.exe`。

## 页面切换

当前没有独立的 Router 类。页面切换通常就是修改页面状态，然后在 `compose` 中按状态声明不同内容。

示例：

```cpp
namespace {
int selectedPage = 0;
}

void compose(core::dsl::Ui& ui, const core::dsl::Screen& screen) {
    ui.row("root")
        .size(screen.width, screen.height)
        .content([&] {
            components::button(ui, "nav.home")
                .size(180.0f, 50.0f)
                .text("Home")
                .onClick([] { selectedPage = 0; })
                .build();

            components::button(ui, "nav.about")
                .size(180.0f, 50.0f)
                .text("About")
                .onClick([] { selectedPage = 1; })
                .build();

            if (selectedPage == 0) {
                ui.text("page.home")
                    .size(320.0f, 48.0f)
                    .text("Home")
                    .fontSize(32.0f)
                    .build();
            } else {
                ui.text("page.about")
                    .size(320.0f, 48.0f)
                    .text("About")
                    .fontSize(32.0f)
                    .build();
            }
        })
        .build();
}
```

页面切换动画目前通过稳定 id 的元素 transition 表达。比如侧边栏指示条只改 `.y(...)`，主体 shell 只改 `.color(...)` / `.opacity(...)` / `.translateX(...)` 等目标属性，Runtime 会从当前值插值到新值。

## DPI 和鼠标坐标

项目区分两个 scale：

- `dpiScale`：窗口内容缩放，用于逻辑尺寸和渲染尺寸换算。
- `pointerScale`：鼠标坐标到 framebuffer 坐标的比例，用于命中测试。

页面 compose 接收的是逻辑尺寸：

```cpp
const float logicalWidth = static_cast<float>(windowWidth) / dpiScale;
const float logicalHeight = static_cast<float>(windowHeight) / dpiScale;
```

所以页面里一般不要直接乘 DPI，也不要自己换算鼠标坐标。Runtime 会处理。

## 脏区与整屏重绘

当前 DSL Runtime 支持 framebuffer cache + scissor 的脏区渲染。

常见局部重绘：

- hover / pressed 颜色变化。
- 按钮按下缩放。
- transition 动画中的位置、透明度、颜色、旋转等变化。
- 文本颜色或透明度变化。

常见整屏或较大范围重绘：

- 窗口尺寸变化。
- DPI 变化。
- 页面切换导致结构变化。
- 元素 id 变化。
- 元素新增或删除。
- 调用 `markFullRedraw()`。

页面和组件不用自己实现脏区，但要做到：

- id 稳定。
- 结构尽量稳定。
- 动画写成 DSL 属性目标值。
- hover / pressed 用 `.states()`。
- 点击只改业务状态，不直接碰 primitive。

## 当前限制

- 还没有独立 Router。
- 还没有 clip / scroll。
- 还没有键盘 focus。
- 还没有 z-index。
- 页面状态目前通常是 cpp 静态变量或业务层变量。
- 结构变化会比纯视觉变化触发更大的重绘范围。
